Graph Layer

TradingAgentsGraph builds ToolNodes, creates LLM clients, wires GraphSetup, then runs graph.invoke() or graph.stream().

Agent Layer

Each create_* factory returns a LangGraph node function that reads and writes fields on AgentState.

Data Layer

LangChain tools call route_to_vendor(), which dispatches to yfinance or Alpha Vantage by category/tool config.

LLM Layer

create_llm_client() chooses OpenAI-compatible, Anthropic, Google, or Azure clients and returns LangChain-compatible chat models.

CLI Layer

cli/main.py collects ticker/date/models/analysts, streams graph chunks, renders progress, logs messages/tools, and writes reports.

Persistence Layer

TradingMemoryLog stores decisions and reflections; checkpointer.py stores LangGraph checkpoints in per-ticker SQLite DBs.

README Picture and Code Reality

The README shows the high-level firm-style process. The source code implements that process as a LangGraph state machine with explicit tool loops and debate loops.

Analyst Team

README: market, social media, news, fundamentals. Code: market, social selector mapped to the renamed Sentiment Analyst, news, and fundamentals.

Research Team

README: bullish and bearish discussion. Code: Bull Researcher and Bear Researcher alternate until 2 * max_debate_rounds.

Risk and Portfolio

README: aggressive, neutral, conservative risk team sends a proposal to manager. Code: three risk debaters alternate until 3 * max_risk_discuss_rounds, then Portfolio Manager emits the final rating.

Tool and Data Source Matrix

Tool wrappers live in tradingagents/agents/utils/*_tools.py. They delegate to tradingagents/dataflows/interface.py.

Confirmed from code: analyst categories are not the same as data vendor categories

GraphSetup.setup_graph() and TradingAgentsGraph._create_tool_nodes() use four analyst/tool-node keys: market, social, news, and fundamentals. The actual vendor-routing categories in TOOLS_CATEGORIES and DEFAULT_CONFIG["data_vendors"] are core_stock_apis, technical_indicators, fundamental_data, and news_data. The social selector now runs the renamed Sentiment Analyst and additionally pre-fetches StockTwits and Reddit outside the vendor router.

Routing Mechanism

route_to_vendor(method, *args, **kwargs) resolves category, reads config, tries primary vendors, and falls back only on AlphaVantageRateLimitError.

Runtime Notes and Code-Level Caveats

These are important when reading the workflow against the current codebase.

Message Cleanup Between Analysts

create_msg_delete() emits RemoveMessage operations for prior messages, then adds a HumanMessage("Continue"). Reports remain in AgentState, but chat history is cleared before the next analyst.

Structured Output Is Decision-Stage Only

Research Manager, Trader, and Portfolio Manager use Pydantic structured output when supported. Analyst, researcher, and risk-debater nodes are natural-language LLM calls.

CLI Streaming Bypasses Some propagate() Hooks

cli/main.py constructs TradingAgentsGraph, then directly streams graph.graph.stream(init_agent_state, **args). That path has its own report saving, but it does not call TradingAgentsGraph.propagate(), so the package-path memory logging, full-state JSON logging, checkpoint recompile/thread-id setup, and checkpoint clearing are not executed there in the current code.

News Tool Registration Detail

The graph-level tools_news ToolNode registers get_insider_transactions, but create_news_analyst() binds only get_news and get_global_news. Since LLM tool calls are constrained by bound tools, insider transactions are not normally requested by the News Analyst.

Outcome Reflection Timing

Decisions are appended as pending. A later same-ticker run resolves prior outcomes by fetching ticker and benchmark history from yfinance and then injecting lessons into the Portfolio Manager prompt.

Final Signal Parsing

SignalProcessor no longer calls an LLM. It parses the **Rating** field from the Portfolio Manager markdown with parse_rating().
